# Copilot Model Switcher

 Copilot Model Switcher 是一款面向 VS Code 的混合模型路由插件，目标是把 GitHub Copilot 当前选择的高能力模型，或用户自定义的第三方高能力模型，用于极低 token 的需求理解、按需进行便宜模型读取结果诊断和最终检查，再把高 token 消耗的上下文读取、工具结果整理、代码生成、长文本输出和调试执行交给用户自购的第三方高性价比模型完成。

当前仓库包含产品需求文档和一个 TypeScript VS Code Extension MVP 骨架。

## 文档

- 产品需求文档位于 docs/PRODUCT_REQUIREMENTS.md，包含核心目标、模型分工、功能模块、技术架构、验收标准、路线图与风险应对。

## 核心定位

- 使用 Copilot 当前选择的模型或第三方高能力模型承担“需求理解/按需读取结果诊断/终检”工作：告诉便宜模型该看哪些文件、该使用哪些受控工具、按什么标准交付；低风险任务会跳过读取结果诊断；其余任务中便宜模型读取上下文后，贵模型再对读取报告做一次诊断；收尾时再做一次最终检查。
- 使用 DeepSeek、Moonshot/Kimi 等第三方 API 承担“执行层”和“中间审查层”工作：上下文读取后的综合、工具结果整理、调试证据归纳、代码生成、长上下文处理、常规补全和修复，以及对便宜模型输出做一次第三方中间审查。
- 所有密钥和路由逻辑保留在本地 VS Code 扩展进程内，第三方 API Key 使用 VS Code SecretStorage 存储。
- 默认把配置保存到当前工作区，多个 VS Code 窗口之间不会再因为全局设置而互相跟随路由模式。

## MVP 功能

- `@model-router` Chat Participant：执行“贵模型理解需求 -> 廉价模型执行”主链路，并在需要时插入“廉价模型读取上下文 -> 贵模型诊断读取结果”，最后再做一次贵模型终检。
- `/debug` 聊天命令：贵模型只理解调试目标、诊断廉价模型读取结果并做终检；廉价模型做调试证据整理和修复建议。
- 中文配置界面：配置 Copilot/第三方贵模型、便宜执行模型、中间审查模型、第三方 API Key、切换性价比模式、查看实时 Token 统计窗口。贵模型、便宜模型和中间审查模型的 token 使用都会自动统计并在重开窗口后恢复，无需额外操作。
- OpenAI-compatible provider 适配：默认内置 DeepSeek 和 Moonshot/Kimi 配置，并支持用户新增或修改任意兼容 provider。

## 关键实现边界

VS Code 官方 API 可以支持扩展注册自有 Chat Participant、调用可用的 Language Model Chat 模型、使用 SecretStorage 管理密钥。插件不应依赖逆向工程或私有接口，也不能把“拦截 GitHub Copilot Chat 内部原始请求”作为稳定合规的 MVP 前提。

## 开发命令

```bash
npm install
npm run compile
```

## 运行和 UI 入口

1. 先在能识别 `npm` 的终端运行 `npm run compile`，确保生成 `out/extension.js`。
2. 在 VS Code 里按 `F5`，选择 `Run Extension` 启动扩展宿主。
3. 扩展启动后可以通过三个入口打开 UI：
	- 状态栏左侧的 `Model Router`。
	- 命令面板中的 `模型路由器：打开配置界面`。
	- 活动栏里的 `Model Router` 侧边栏视图。

如果需要 F5 前自动编译，选择 `Run Extension With Build`；该配置会执行 `.vscode/tasks.json` 中的 `npm: compile` 任务。

## Copilot Chat 内切换贵模型

在 Copilot Chat 中使用 `@model-router` 时，插件会在回复顶部显示当前聊天模型。性价比模式开启后，贵模型会自动跟随用户当前在 Copilot Chat 中选择的模型，但只允许做需求理解、按需读取结果诊断和一次终检。

也可以直接输入：

```text
@model-router /useCurrentModel
```

如果在配置 UI 中手动保存固定 Planner/Judge 模型，则会关闭性价比模式，改为固定贵模型。

## 性价比模式

VS Code 官方扩展 API 目前不能把自定义按钮直接插入 GitHub Copilot Chat 自带的模型选择下拉菜单，也不能替换该下拉菜单的内部行为。插件采用合规替代方案：提供 `性价比模式` 路由模式。

 开启后：

- 贵模型自动跟随 Copilot Chat 当前选择的模型。
- 贵模型只做低 token 需求理解、按需进行便宜模型读取结果诊断和收尾终检，不生成代码、不长篇解释、不参与多轮改写，也不为自己读取项目文件或调用工具。
- 如果配置了中间审查模型，则会在便宜模型完成后先用第三方审查模型检查结果是否满足贵模型要求，再决定是否进入贵模型终检。
- 低风险、直接型任务会自动跳过一次贵模型读取结果诊断，以进一步降低贵模型消耗并减少延迟。
- 如果理解需求时需要文件、诊断、搜索或命令结果，贵模型只把这些需求写进便宜模型的文件/工具计划。
- 代码生成、调试证据整理、长文本输出、常规修复和自检主体都交给配置的廉价执行模型。
- 状态栏会显示 `性价比模式`，配置 UI 底部会显示 `已启用性价比模式`。
- 配置 UI 提供实时 Token 统计窗口，自动显示贵模型、便宜模型与中间审查模型的本次/会话 token 使用情况，并恢复最近一次工作区统计结果，无需额外操作。

## 第三方贵模型与中间审查模型

- 当配置为 `第三方 API` 贵模型来源时，需求理解、读取结果诊断和终检都会改走用户自定义的第三方 provider/model，不再依赖 Copilot 当前模型。
- 中间审查模型默认可配置为如 `deepseek-v4-pro` 一类的中档模型，用于检查便宜模型输出是否满足贵模型给出的验收要求。
- 第三方 provider 列表支持在配置面板内新增、编辑、测试连接和保存；API Key 仍然只保存在 VS Code SecretStorage。

入口：

- 状态栏 `性价比模式` / `模型路由器`。
- 命令面板：`模型路由器：启用性价比模式`。
- Copilot Chat：`@model-router /valueMode`。
- `@model-router` 回复顶部的 `启用性价比模式` 按钮。
